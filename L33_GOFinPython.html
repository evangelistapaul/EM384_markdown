Goodness of Fit in Python¶

by COL Paul Evangelista¶

2 August 2022¶

Given a sample of data from a random process, it is often desirable to determine whether or not the data fits a known distribution. If we can assume a reasonable goodness of fit to a known distribution, simulation of the random process with pseudo-random variables becomes possible. Simulation provides an opportunity for analysis and insights of how the process behaves and integrates with other aspects of its environment. Lets start with an example using the IOCT times from a recent class of USMA cadets. We will begin with a plot of the empirical distribution function (ECDF).

Notes
-----
The probability density function for `norm` is:

.. math::

f(x) = \frac{\exp(-x^2/2)}{\sqrt{2\pi}}

for a real number :math:`x`.

The probability density above is defined in the "standardized" form. To shift
and/or scale the distribution use the ``loc`` and ``scale`` parameters.
Specifically, ``norm.pdf(x, loc, scale)`` is identically
equivalent to ``norm.pdf(y) / scale`` with
``y = (x - loc) / scale``. Note that shifting the location of a distribution
does not make it a "noncentral" distribution; noncentral generalizations of
some distributions are available in separate classes.

Examples
--------
>>> from scipy.stats import norm
>>> import matplotlib.pyplot as plt
>>> fig, ax = plt.subplots(1, 1)

Calculate the first four moments:


>>> mean, var, skew, kurt = norm.stats(moments='mvsk')

Display the probability density function (``pdf``):

>>> x = np.linspace(norm.ppf(0.01),
... norm.ppf(0.99), 100)
>>> ax.plot(x, norm.pdf(x),
... 'r-', lw=5, alpha=0.6, label='norm pdf')

Alternatively, the distribution object can be called (as a function)
to fix the shape, location and scale parameters. This returns a "frozen"
RV object holding the given parameters fixed.

Freeze the distribution and display the frozen ``pdf``:

>>> rv = norm()
>>> ax.plot(x, rv.pdf(x), 'k-', lw=2, label='frozen pdf')

Check accuracy of ``cdf`` and ``ppf``:

>>> vals = norm.ppf([0.001, 0.5, 0.999])
>>> np.allclose([0.001, 0.5, 0.999], norm.cdf(vals))
True

Generate random numbers:

>>> r = norm.rvs(size=1000)

And compare the histogram:

>>> ax.hist(r, density=True, histtype='stepfilled', alpha=0.2)
>>> ax.legend(loc='best', frameon=False)
>>> plt.show()

parameters = dist.fit(x)
#see the parameters
print(parameters)

(3.0576666666900003, 0.5723121719890853)

In the tuple above, 3.05767 is the estimated mean or location, and 0.5723 is the estimated scale or standard deviation. Location and scale are common terms used when referencing distributions. Look at the doc string and notice how location and scale are both described in the very first part of the string. The next command uses the Kolmogorov-Smirnov test to check for goodness of fit. The null hypothesis is that the data comes from a normal distribution with parameter estimates as described in the first sentence of this paragraph. The alternative hyphothesis is that the data does not come from a normal distribution with these parameter values.

#try to find location of max difference between ecdfs, which is the sample statistic
maxd = 0
xloc = 0
exloc = 0
ensloc = 0

for i in np.linspace(1,5,100000):
    d = abs(ex(i)-ens(i))
    if(d > maxd):
        maxd = d
        xloc = i
        exloc = ex(i)
        ensloc = ens(i)

print("estimated KS test statistic = ", maxd, "; x location = ", xloc)

estimated KS test statistic =  0.1626700000000001 ; x location =  3.083340833408334

Notice how closely the estimated KS statistic above approximates the KS statistic calculated with stats.kstest(). The KS statistic is actually a very simple test. Given two ECDFs, move along the space of the random variable (x-axis) and find the maximum difference between the ECDF values. The magnitude of this max difference is the KS statistic. The plot below shows a black dashed line where the maximum difference between ECDFs exists. The maximum difference between two ECDFs from the same distribution follows a known distribution, and when this difference exceeds a certain threshold, we can use the principles of inferential statistics to declare that it is unlikely the two ECDFs come from the same distribution. This test is also non-parametric, which is a favorable and powerful property. This means that the parametric properties of the underlying distributions do no matter.

.. currentmodule:: scipy.stats

This module contains a large number of probability distributions,
summary and frequency statistics, correlation functions and statistical
tests, masked statistics, kernel density estimation, quasi-Monte Carlo

Statistics is a very large area, and there are topics that are out of scope
for SciPy and are covered by other packages. Some of the most important ones

also covered by ``scipy.stats``.
- `Pandas `__: tabular data, time series
functionality, interfaces to other statistical languages.
- `PyMC3 `__: Bayesian statistical
modeling, probabilistic machine learning.

Some of these distributions are a bit non-standard, so it is always worthwhile to research the distributions before using. Let's consider the top choice: "burr12". I never heard of a "burr12" distribution until I wrote this tutorial. The Burr-12 distribution is a non-negative distribution commonly used to model household income. This should tell you that there is a long positive tail. There are two parameters for the Burr-12 distribution, c and d, both of which are used to calculate the mean and variance of the distribution. The details of this distribution are beyond the scope of this course, but let's see how well it fits our IOCT data.

The Burr-12 distribution does seem to fit our IOCT data well. We have no statistical reason to reject the null hypothesis.

We conducted two GOF tests in this analysis. For test 1, this was the null hypothesis($H_o$): The sample IOCT data come from a normal distribution with aforementioned parameter values. This was the alternative hypothesis($H_a$): The sample IOCT data do not come from a normal distribution with aforementioned parameter values. Based on a p-value of 0.009, we would reject the null hypothesis. We have statistical evidence to support the conclusion that the underlying distribution is not normal.

For test 2, this was the null hypothesis($H_o$): The sample IOCT data come from a Burr-12 distribution with aforementioned parameter values. This was the alternative hypothesis($H_a$): The sample IOCT data do not come from a Burr-12 distribution with aforementioned parameter values. Based on a p-value of 0.96, we fail to reject the null hypothesis. There is no evidence to reject our use of the Burr-12 distribution to represent IOCT data.

When drawing conclusions from statistical tests, notice that we never say that we "accept the alternative hypothesis." It is correct to only say that you either "fail to reject the null hypothesis" or you "reject the null hypothesis." This is one of the many subtle but important uses of phrases and words in statistics.

Two exercises are left the reader: 1) attempt to approximate the KS statistic using a large sample from a Burr-12 distribution. We conducted a similar exercise for the normal distribution using a for loop and finding the max of the absolute difference above. 2) choose another distribution with a high p-value and test the fit using the KS test; visualize the fit by plotting the ECDFs.

What if the data does not fit any distributions?¶

It is possible that collected data will not fit any parametric distribution well. Simulation of parametric distributions is often not the best method to represent a random process. Given a sufficient sample, bootstrapping is a method that can be used to simulate a random process. Lets use the IOCT data again and suppose there simply is no good fit for this data. Bootstrapping involves simply randomly sampling from the existing data, typically with replacement:
